用我自己能看懂的语言，简单回顾一下我的Github+Hexo的建(cai)站(keng)经历，以备后患。

以下是碎碎念：

  1. 看官方文档
  2. 粗心犯大错
  3. 先理解原理
  4. 熟悉简单语法
  5. Hugo对我来说是个坑…🤣

安装

  • Node.js
  • Git

安装完两个程序后，打开cmd指令，使用npm安装Hexo

$ npm install -g hexo-cli

建库

在github上建立一个仓库，名称为 githubname.github.io
打开Git bash，输入

域名

到域名注册网站申请好域名并且配置好参数，在仓库的setting中把域名放到domain

建站

执行以下命令，Hexo将在你的博客所在的文件夹中新建一个网站和系列文件

$ hexo init [foldername]

$ cd [foldername]

$ npm install

下面这个是安装Hexo的服务器和部署独立模块。

$ npm install hexo-server –save

$ npm install hexo-deployer-git –save

修改配置

config.yml 网站的配置信息

参数 描述 值
language 语言 zh-Hans
permalink 文章的永久链接格式 :year/:month/:day/:title/
url 文章存放的目录 默认值
theme 主题 主题名称

deploy:
type: git
repo: 仓库的http地址，结尾是.git
branch: master

theme主题文件夹的_config.yml是主题的配置信息；
package.json应用程序的信息；
scaffolds 模板文件夹，新建文章时默认填充的内容
source 资源文件夹，posts文件夹的md文件将会被解析并放到public文件夹；
在source文件夹中创建一个无后缀的CNAME文件，内容填上域名地址即可。

Hexo命令

$ hexo init [foldername]

新建一个博客。

$ hexo new [layout] [title]

新建一篇文章。如果没有设置 layout 的话，默认使用config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。
layout布局有三种：post、page和draft。
他们分别对应不同路径，自定义的布局将储存到source/posts文件夹中。

$ hexo generate

生成静态文件，可缩写 hexo g。

$ hexo publish [layout] [filename]

发布草稿。

$ hexo server

启动服务器。默认情况下访问网址 http://localhost:4000/
hexo s-p 5000 重设端口。

$ hexo deploy

部署网站，可简写hexo d，部署之前需要用hexo g先生称静态文件。

$ hexo render [file1] [file2] …

渲染文件（暂时没有用过）。

$ hexo clean

清除缓存文件 (db.json) 和已生成的静态文件 (public)。
在某些情况（尤其是更换主题后），如果发现对站点的更改无论如何也不生效，需要运行该命令。

其他碎碎念

  1. 与其看凌乱的博客，不如去看官方文档，尽管有的是英文看起来有点麻烦，但至少逻辑清晰且准确。
  2. 粗心犯大错。一个符号就导致了全部文章GG_(:з)∠)_ （config里的永久链接写少了一个/，导致所以文章都Undefined）。
  3. 先明白整个框架和原理，然后再去实践。理解是多么重要。
  4. 熟悉语法。因为cmd指令和md语法忘了的缘故，很多都是边查边用的。
  5. 不要盲目模仿。因为文章一直出不来，就想换hugo搭博客，但hugo主要是给Linux系统设计的，Windows需要改环境变量等等，反而更麻烦了。另外网上教程语法全是Linux的，哭瞎。
